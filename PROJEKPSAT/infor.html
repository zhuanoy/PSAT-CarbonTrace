Dekomposisi Masalah

Arsitektur kalkulator karbon — Input · Proses · Output

Dekomposisi adalah teknik memecah masalah besar menjadi bagian-bagian kecil yang lebih mudah diselesaikan. Kalkulator jejak karbon dibagi menjadi tiga sub-komponen utama yang saling terhubung:

📥

Input

  • Penggunaan BBM (liter/hari)
  • Konsumsi listrik (kWh/bulan)
  • Jarak transportasi (km/hari)
  • Jenis & frekuensi makanan
  • Moda transportasi
⚙️

Proses

  • Kalkulasi emisi per kategori
  • Jumlah total CO₂e (kg)
  • Validasi nilai input (> 0)
  • Klasifikasi kategori emisi
  • Penentuan saran tindakan
📤

Output

  • Total emisi (kg CO₂e)
  • Kategori: Rendah / Sedang / Tinggi
  • Grafik breakdown emisi
  • Saran praktis pengurangan
  • Perbandingan rata-rata global
💡 Dengan memisahkan sistem menjadi tiga komponen, setiap bagian dapat dikembangkan, diuji, dan diperbaiki secara independen — meningkatkan keterbacaan kode dan kemudahan pemeliharaan.

Logika Percabangan (If-Else)

Alur kondisi untuk mengklasifikasikan tingkat emisi pengguna

Logika percabangan If-Else digunakan untuk memilah total emisi pengguna ke dalam 4 kategori khusus berdasarkan ambang batas (threshold) yang telah ditentukan. Setiap kategori memberikan respons dan saran yang berbeda.

Kondisi Kategori Rentang (kg CO₂e/hari) Saran Utama
total < 5 🟢 Rendah < 5 kg Pertahankan gaya hidup ramah lingkungan
total < 15 🟡 Sedang 5 – 14 kg Kurangi penggunaan kendaraan pribadi
total < 30 🔴 Tinggi 15 – 29 kg Beralih ke energi terbarukan & transportasi publik
total >= 30 🔥 Sangat Tinggi ≥ 30 kg Audit menyeluruh gaya hidup & konsultasi ahli
// Pseudocode — Logika Percabangan Kategori Emisi
IF total < 5 THEN
    kategori = "Rendah"
    saran    = "Pertahankan gaya hidup ramah lingkungan 🌿"
ELSE IF total < 15 THEN
    kategori = "Sedang"
    saran    = "Kurangi kendaraan pribadi, gunakan transportasi umum 🚌"
ELSE IF total < 30 THEN
    kategori = "Tinggi"
    saran    = "Beralih ke energi terbarukan ☀️"
ELSE
    kategori = "Sangat Tinggi"
    saran    = "Audit menyeluruh gaya hidup & konsultasi ahli 🔥"
END IF
💡 Struktur IF-ELSE IF-ELSE menjamin hanya satu cabang yang dieksekusi. Urutan kondisi dari nilai terkecil ke terbesar adalah kunci agar tidak terjadi kesalahan klasifikasi.

Pemetaan Tipe Data

Tipe data yang tepat untuk setiap variabel input kalkulator

Pemilihan tipe data yang tepat sangat penting agar kalkulasi berjalan akurat dan efisien. Tipe data yang salah dapat menyebabkan error pembulatan, overflow, atau hasil yang tidak valid.

Variabel Input Contoh Nilai Tipe Data Alasan
Penggunaan BBM 3.5 liter/hari float Nilai desimal diperlukan untuk akurasi emisi
Konsumsi Listrik 150 kWh/bulan float Bisa mengandung desimal (150.5 kWh)
Jarak Transportasi 12 km/hari float Jarak bisa pecahan (12.5 km)
Jenis Makanan "daging", "nabati" string Kategori teks, bukan angka
Frekuensi Daging 3 kali/minggu int Bilangan bulat (tidak ada 1.5 kali)
Moda Transportasi "motor", "bus" string Pilihan kategori teks
Total Emisi CO₂e 12.847 kg float Hasil kalkulasi memerlukan presisi desimal
Kategori Emisi "Sedang" string Label hasil klasifikasi If-Else
⚠️ Gunakan float untuk semua nilai numerik yang mewakili pengukuran fisik. Hindari int untuk nilai kalkulasi emisi karena pembulatan dapat menyebabkan ketidakakuratan signifikan dalam hasil akhir.

Validasi Alur — Trace Table Testing

Mendeteksi potensi infinite loop sebelum kode dijalankan

Trace table adalah teknik pengujian manual dengan mensimulasikan eksekusi program baris per baris, mencatat nilai variabel di setiap langkah. Tujuannya mendeteksi infinite loop — kondisi di mana program terus berulang tanpa pernah berhenti — sebelum kode diterjemahkan ke bahasa pemrograman.

Iterasi BBM (L) Listrik (kWh) Transport (km) Total CO₂e (kg) Total > 0? Status
1 0 0 0 0.00 ❌ Tidak ↩ Input Ulang
2 -2 0 5 Error ❌ Tidak ↩ Input Ulang
3 2 100 10 8.74 ✅ Ya ✔ Lanjut → Klasifikasi
4 5 200 30 21.30 ✅ Ya ✔ Lanjut → Kategori Tinggi
5 0.1 0 0 0.23 ✅ Ya ✔ Lanjut → Kategori Rendah

🔄 Potensi Infinite Loop & Pencegahannya

❌ Risiko Infinite Loop
WHILE total == 0 DO
  // Jika user terus
  // input 0, loop
  // tidak pernah berhenti!
  input()
END WHILE
✅ Solusi Pencegahan
maxCoba = 3
coba = 0
WHILE total == 0 AND coba < maxCoba
  input()
  coba += 1
END WHILE
🛡️ Selalu tambahkan counter batas percobaan (maxTry) pada setiap loop input. Ini memastikan program tidak terjebak jika pengguna terus memasukkan nilai tidak valid, sekaligus memberikan pesan error yang informatif.
